How these numbers line up with Polymarket

P&L sums the same per-position dollar gain/loss that Polymarket shows on each row of the Positions tab: (mark − entry) × shares, with mark = mid price for open trades and the realised close price for closed trades. Add up the +/− column on Polymarket and you get this number. It excludes wallet-level effects (deposits, withdrawals, gas, builder credits) so it reflects only what the agent did. Cross-check anytime on the Polymarket UI or the on-chain transfer log.

Rows with result badge SETTLE are markets Polymarket has already resolved (official close_price of $0 or $1 per share). Those closes are written into this ledger automatically each run so win rate and headline P&L stay aligned with Polymarket even when the CLOB order book is gone and the agent could not place a final sell.
